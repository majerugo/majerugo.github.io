## Remote Code Execution

This website runs an *XWiki instance*, and we identify the version as `XWiki Debian 15.10.8`. This version is vulnerable to remote code execution (RCE) through Groovy code injection via the `text` parameter in `/xwiki/bin/get/Main/SolrSearch?media=rss&text=`.

Use this [GitHub repository](https://github.com/dollarboysushil/CVE-2025-24893-XWiki-Unauthenticated-RCE-Exploit-POC) to exploit the vulnerability.

This gives us a reverse shell on the server as user `xwiki`.

## First Privilege Escalation

We then look for credentials for user `oliver` in the wiki files. In `/usr/lib/xwiki/WEB-INF/hibernate.cfg.xml`, we find a password for the database user `oliver`. This behavior is documented here: [XWiki-Installation-MySQL](https://www.xwiki.org/xwiki/bin/view/Documentation/AdminGuide/Installation/InstallationWAR/InstallationMySQL/).

Trying this password over SSH works.

## Final Privilege Escalation

We use `linpeas.sh` to identify a path to root. We find multiple SUID binaries that are not flagged by `linpeas.sh`.
One of them is `ndsudo`, which allows privilege escalation to root.
Set the `PATH` environment variable to include a binary that spawns a root shell when executed by `ndsudo`.
Then trigger `ndsudo` with the `nvme-list` argument.

See this GitHub repository for details: [CVE-2024-32019-POC](https://github.com/AzureADTrent/CVE-2024-32019-POC).
This gives us a root shell.
